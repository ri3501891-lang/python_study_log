# Study Log CLI

Pythonで作成した、学習内容を記録・管理するためのCLIアプリです。  
ターミナル上で学習ログを追加・一覧表示・検索・集計できるようにし、SQLを使わずにJSONファイルへ保存する構成で実装しました。

## 概要

このアプリは、日々の学習内容を簡単に記録することを目的とした学習用ミニプロジェクトです。  
ファイル操作、JSONの読み書き、関数分割、例外処理、CLIアプリの構成を学ぶために作成しました。

## 主な機能

- 学習ログの追加
- 学習ログの一覧表示
- 科目・トピックによる検索
- 直近7日間の学習時間の集計
- JSONファイルへの保存と読み込み

## 使用技術

- Python
- JSON
- pathlib
- datetime

## ファイル構成

```text
study_log_cli/
├─ main.py
└─ logs.json
```

- `main.py` : アプリ本体
- `logs.json` : 学習ログを保存するJSONファイル



## 操作イメージ

```text
=== 学習ログCLI ===
1. 学習ログを追加する
2. 学習ログを一覧表示する
3. 科目で検索する
4. 直近7日間の学習時間を表示する
5. 終了する
```



## 学習メモ

### 日付の扱い（`datetime.strptime()`）

直近7日間の学習時間を集計するために、JSONに保存された日付文字列を `datetime` 型に変換して扱う方法を学んだ。

```python
d = datetime.strptime(log["date"], "%Y-%m-%d").date()
```

- `log["date"]`  
  JSONに保存されている日付文字列を取得している。
- `datetime.strptime(..., "%Y-%m-%d")`  
  文字列を「年-月-日」形式としてパースし、`datetime` 型に変換している。
- `.date()`  
  時刻情報を切り捨てて、日付だけを取り出している。

まだ `datetime` 型の感覚は完全には掴めていないため、今後は他のフォーマットやタイムゾーンも含めて理解を深めたい。

### 例外処理と `continue`

壊れた日付データでアプリ全体が落ちないように、例外処理と `continue` を使う実装を試した。

```python
try:
    d = datetime.strptime(log["date"], "%Y-%m-%d").date()
except ValueError:
    continue
```

- `try` / `except ValueError`  
  日付文字列の形式が想定外だった場合でも、アプリ全体は止めずに処理を続行するため
- `continue`  
  エラーになったログだけをスキップし、次のログの処理へ進める。

最初は `continue` の挙動がイメージしづらかったが、「壊れた1件だけ飛ばして、ループ自体は続ける」という動きをしていると理解できた。

### `dict.get()` による安全なアクセス

ログの `minutes` を集計する際に、`dict.get()` を使って「壊れたデータでも落ちにくい」書き方を学んだ。

```python
total += log.get("minutes", 0)
```

- `log["minutes"]` だと、キーが存在しない場合に `KeyError` でアプリが止まる。
- `log.get("minutes", 0)` を使うことで、
  - キーがあれば値を返す
  - なければデフォルト値として `0` を返す

という動きになり、「不完全なデータが混ざっていても集計処理は継続する」という防御的な実装になると理解した。

### エントリーポイントの分離（`if __name__ == "__main__":`）

```python
if __name__ == "__main__":
    main()
```

この書き方を使うことで、`python main.py` で直接実行したときだけメインメニューを起動し、他ファイルから `import` したときには CLI が自動で動かない構成にできると学んだ。
